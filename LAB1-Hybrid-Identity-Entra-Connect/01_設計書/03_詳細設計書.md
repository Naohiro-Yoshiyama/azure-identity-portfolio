## 1. 対象システム構成詳細

### 1.1 構成概要

- 単一フォレスト／単一ドメイン構成  
- サインイン方式：Password Hash Synchronization（PHS）  
- Seamless SSO：有効  
- OUフィルタ：SyncUsers OU のみ同期  
- Active / Staging 冗長構成（Activeは常に1台）  
- カスタム同期ルールなし（既定同期ルールを使用）  

### 1.2 サーバ構成

| サーバ名 | 役割 |
| -------- | ---- |
| addc01 | Active Directory Domain Services / DNS |
| mec01 | Microsoft Entra Connect (Active) |
| mec02 | Microsoft Entra Connect (Staging) |
| app01 | 検証端末 |

詳細なOS・IP・役割設定は「3.サーバ詳細設計」を参照。

## 2. Active Directory 詳細設計

### 2.1 フォレスト／ドメイン

| 項目 | 規定値 |
| ---- | ------ |
| フォレスト数 | 1 |
| ドメイン数 | 1 |
| ドメインFQDN | corp.local |
| NetBIOS名 | CORP |
| フォレスト機能レベル | Windows Server 2016 以上 |
| ドメイン機能レベル | Windows Server 2016 以上 |
| FSMO配置 | addc01 |
| グローバルカタログ | 有効 |

### 2.2 UPN設計

| 項目 | 設定値 |
| ---- | ------ |
| 規定UPNサフィックス | corp.local (ADドメインFQDN) |
| 追加UPNサフィックス | corp.example.com (ルーティング可能) |
| 使用UPNサフィックス | corp.example.com (ユーザー割当用) |

> 注記：corp.localはnon-routable ドメインのため、Entra IDとの整合性を考慮し、ルーティング可能なcorp.example.comをユーザー割当用サフィックスとして使用する。

### 2.3 OU設計(同期対象)

| 項目 | 設定値 |
| ---- | ------ |
| 同期対象OU | OU=SyncUsers,DC=corp,DC=local |
| 同期対象オブジェクト | User / Group |
| 同期対象外 | 上記以外すべて |

## 4. Microsoft Entra Connect 詳細設定

### 4.1 Azure 接続設定

| 項目 | 設定値 |
| ---- | ------ |
| 接続方式 | 既存テナントへ接続 |
| Azure AD テナント名 | `<確認中>` |
| 使用権原 | Global Administrator (初期設定時のみ) |

### 4.2 サインイン方式

| 項目 | 設定値 |
| ---- | ------ |
| Singn-in method | Password Hash Synchronization (PHS) |
| Seamless SSO | 有効 |
| ADFS | 未使用　(導入しない) |

### 4.3 同期対象オブジェクト

| オブジェクト | 同意有無 | 補足 |
| ------------ | ------- | ---- |
| User | 有効 | SyncUsers OU 配下のみ |
| Group | 有効 | SyncUsers OU 配下のみ |
| Contact | 無効 | 対象外 |
| Device | 無効 | 対象外 |

> OUフィルタ適用により、SyncUsers OU 外のGroupは同期されない。

### 4.4 OUフィルタ設定

| 項目 | 設定値 |
| ---- | ------ |
| OU Filtering | 有効 |
| 選択OU | OU=SyncUsers,DC=corp, DC=local |
| 非選択OU | 上記以外すべて |

### 4.5 同期オプション

| 項目 | 設定値 | 設計意図 |
| ---- | ------ | -------- |
| Password Writeback | 無効 | 対象外 |
| Group Writeback | 無効 | 対象外 |
| Device Writeback | 無効 | 対象外 |
| Exchange Hybrid | 未構成 | 対象外 |
| Deletion Threshold | 500 (規定値) | 誤削除防止のため規定値を採用 |

### 4.6 Scheduler設定

| 項目 | mec01 (Active) | mec02 (Staging) |
| ----- | -------------- | -------------- |
| SyncCycleEnable | True | True |
| StagingModeEnabled | False | True |
| 同期間隔 | 30分 (規定) | 30分 (規定 ) |

## 5. ネットワーク詳細設計

### 5.1 必要通信

| 通信元 | 通信先 | プロトコル | ポート | 用途 |
| ------ | ------ | --------- | ----- | ---- |
| mec01/mec02 | addc01 | TCP | 389 | LDAP |
| mec01/mec02 | addc01 | TCP | 636 | LDAPS |
| mec01/mec02 | addc01 | TCP | 3268 | Global Catalog |
| mec01/mec02 | Entra ID | TCP | 443 | Import / Sync |
| app01 | Entra ID | TCP | 443 | サインイン検証 |

>注記：mec02 (Staging)もImport / Sync のため、インターネット (TCP 443) 通信が必須。

### 5.2 通信方向

- mec01 : AD ⇔ Entra ID (Import / Sync / Export)
- mec02 : AD ⇔ Entra ID (Import / Sync、Exportなし)
- addc01 : DNS応答 / LDAP提供

### 5.3 セキュリティ要件

| 項目 | 設定値 |
| ---- | ------ |
| TLS | 1.2以上 |
| Proxy | 未使用 |
| インターネット接続 | 直接接続 (Fw制御) |

## 9. ログ・監視設計

### 9.1 取得対象

| 種別 | 取得元 | 目的 |
| ---- | ------ | ---- |
| 同期履歴 | Synchronization Service Manager | 成否確認 |
| Windowsイベント | Application | 障害検知 |
| クラウド状態 | Entra 管理センター | 同期状態確認 |

### 9.2 Microsoft Entra Connect Health (拡張想定)

本設計では、同期基盤の健全性監視を目的として
Microsoft Entra Connect Health の導入を前提とする。

#### 9.2.1 導入前前提条件

| 項目 | 内容 |
| ---- | ---- |
| 必要ライセンス | Microsoft Entra ID P1 以上 |
| 導入対象 | mec01 / mec02 |
| 接続先 | Microsoft Entra 管理センター |

#### 9.2.2 監視対象項目

| 区別 | 監視内容 |
| ---- | ------- |
| 同期状態 | Success / Failure |
| Export 状態 | エラー有無 |
| 削除監視 | Deletion Threshold 超過 |
| パフォーマンス | 同期処理時間 / CPU / メモリ |
| AD 接続 | LDAP / GC 接続状態 |

#### 9.2.3 アラート設計 (想定)

| 項目 | 条件 | 対応方針 |
| ---- | ---- | ------- |
| 同期失敗 | 連続失敗 | 原因調査・Delta再実行 |
| 大量削除 | Threshold超過 | Export停止確認 |
| Exportエラー | 失敗検知 | Connector Space確認 |

---

#### 9.2.4 実装状況

※　本LAB環境ではライセンス制約により未実装とする。
本番環境では Connect Health Agent を各同期サーバへ導入する。
