# AD構築手順書（corp.local / addc01・addc02）

対象：検証環境（vSphere 上の Windows Server 2022）で **Active Directory Domain Services（AD DS）＋ DNS** を 2台冗長（DC1/DC2）で構築する。

---

## 0. 前提（設計値）

### 0.1 サーバ／IP（固定）
| サーバ | 役割 | IP | GW |
|---|---|---:|---:|
| addc01 | DC1 / DNS | 172.16.1.50 | 172.16.1.1 |
| addc02 | DC2 / DNS | 172.16.1.49 | 172.16.1.1 |

### 0.2 DNS参照（設計方針）
- **DC自身は「自身（優先）」＋「相手DC（代替）」** を基本とする（推奨）

### 0.3 ADドメイン（新規フォレスト）
- ドメインFQDN：`corp.local`
- NetBIOS：`CORP`
- 機能レベル：Windows Server 2016 以上

### 0.4 UPN / OU（後続の Entra Connect 同期前提）
- 追加UPNサフィックス：`corp.example.com`（ユーザー割当用）
- OU：`OU=SyncUsers` / `OU=Admins` / `OU=ServiceAccounts` を作成

---

## 1. 作業前チェックリスト

- [ ] addc01 / addc02 に Windows Server 2022 がインストール済み
- [ ] 同一ネットワークで相互疎通できる（IP/Mask/GW 設定済み）
- [ ] ローカル管理者でログオンできる
- [ ]（推奨）スナップショットは「構築フェーズのみ」使用（運用では恒常利用しない）

## 2. addc01（DC1）構築手順

### 2.1 ホスト名変更（addc01）
1. addc01 にログオン
2. **Server Manager** → **Local Server**
3. **Computer name** → **Change**
4. Computer name：`addc01`
5. 再起動

**確認（任意 / PowerShell）**
```powershell
hostname
```

### 2.2 IPアドレス（固定）設定（addc01）

1. **Server Manager** → **Local Server** → **Ethernet** をクリック  
2. **Change adapter options** → 対象 NIC を右クリック → **Properties**  
3. **Internet Protocol Version 4 (TCP/IPv4)** → **Properties**  
4. 下記を設定  
   - IP address：`172.16.1.50`  
   - Subnet mask：`255.255.255.0`（/24）  
   - Default gateway：`172.16.1.1`  
   - Preferred DNS server：`172.16.1.50`（自身）  
   - Alternate DNS server：`172.16.1.49`（addc02）  

**確認（任意 / PowerShell）**
```powershell
ipconfig /all
```

### 2.3 AD DS / DNS 役割インストール（addc01）

1. **Server Manager** → **Manage** → **Add Roles and Features**
2. **Role-based or feature-based installation**
3. 対象サーバ：`addc01`
4. **Roles**
   - ✅ **Active Directory Domain Services**
   - ✅（促されたら）**DNS Server** も追加
5. **Features**：既定のまま
6. **Install**

---

### 2.4 ドメインコントローラ昇格（新規フォレスト作成）

1. **Server Manager** の通知フラグ → **Promote this server to a domain controller**
2. **Deployment Configuration**
   - ✅ **Add a new forest**
   - Root domain name：`corp.local`
3. **Domain Controller Options**
   - Forest functional level：Windows Server 2016 以上
   - Domain functional level：同上
   - ✅ DNS（通常チェック済み）
   - ✅ Global Catalog（通常チェック済み）
   - （任意）RODC：OFF
   - DSRM パスワード：任意に設定
  > 📝 **DSRMパスワードの管理方針**
  > - ドメイン管理者パスワードとは**別のパスワード**を設定すること
  > - 設定後はパスワード管理ツール等に即時保管すること
  > - addc01 / addc02 は**同一のDSRMパスワード**を設定する（運用管理の統一）
  > - DSRMはAD障害時の復旧手段であり、紛失した場合の復旧は困難
4. **Additional Options**
   - NetBIOS：`CORP`（自動提案されることが多い）
5. **Paths**
   - 既定のまま（検証なら通常OK）
6. **Prerequisites Check** → **Install**
7. 自動的に再起動

---

## 2.5 初期確認（addc01）

ログオン後、以下を確認：

### (A) ドメイン参加状態
- `CORP\Administrator` などでログオンできること

### (B) DNSゾーン作成（GUI）

1. **Tools** → **DNS**
2. **Forward Lookup Zones** に `corp.local` があること

### (C) DC診断（推奨）
```powershell
dcdiag /v
```

### (D) SYSVOL/NETLOGON 共有
```powershell
net share | findstr /I "SYSVOL NETLOGON"
```

## 2.6 DNSフォワーダー設定（addc01）

Entra Connect はインターネット（`login.microsoftonline.com` 等）へ
TCP 443 で通信するため、DNS がインターネット名前解決できる必要がある。
ルートヒント（既定）でも動作するが、応答速度・管理性の観点から
フォワーダーの明示設定を推奨する。

---

### 手順（GUI）

1. **Server Manager** → **Tools** → **DNS**
2. 左ペインで `addc01` を右クリック → **Properties**
3. **Forwarders** タブ → **Edit**
4. 以下のいずれかを入力して **OK**

| 用途 | IPアドレス |
|------|-----------|
| Google Public DNS | `8.8.8.8` |
| Google Public DNS（代替） | `8.8.4.4` |

> 📝 本番環境では社内プロキシや指定DNSサーバを使用すること。
> 本LABでは Google Public DNS を使用する。

---

### 手順（PowerShell）
```powershell
Add-DnsServerForwarder -IPAddress "8.8.8.8","8.8.4.4"
```

---

### 動作確認
```powershell
# フォワーダー設定確認
Get-DnsServerForwarder

# インターネット名前解決確認
Resolve-DnsName login.microsoftonline.com
```

期待値：IPアドレスが返ること（NXDOMAIN や タイムアウトでないこと）

---

> ⚠️ addc02 はレプリケーションによりフォワーダー設定が自動反映される。
> 反映されない場合は addc02 でも同手順を実施すること。

# 3. addc02（DC2）構築手順

ポイント：**ドメイン参加／昇格前は DNS を addc01 に向ける**（ドメイン探索のため）。  
昇格後に **「自身優先 + 相手代替」** に戻す。

---

## 3.1 ホスト名変更（addc02）

1. addc02 にログオン
2. **Server Manager** → **Local Server** → **Computer name** → **Change**
3. `addc02` に変更 → 再起動

---

## 3.2 IPアドレス（固定）設定（addc02：昇格前）

1. IPv4 設定画面を開く
2. 下記を設定  
   - IP address：`172.16.1.49`
   - Subnet mask：`255.255.255.0`
   - Default gateway：`172.16.1.1`
   - Preferred DNS server：`172.16.1.50`（addc01）※昇格前
   - Alternate DNS server：空欄でも可（昇格後に設定）
  
  > ⚠️ **昇格前の一時設定**：addc02 がドメイン `corp.local` を検索するために、
> 昇格前は addc01 の DNS を参照する必要がある。
> 昇格完了後は **3.5節** の手順で「自身優先 ＋ 相手DC代替」に変更すること。

**疎通確認**
```powershell
ping 172.16.1.50
```

## 3.3 AD DS / DNS 役割インストール（addc02）

1. **Server Manager** → **Manage** → **Add Roles and Features**
2. 役割：✅ **AD DS**、✅ **DNS Server**
3. **Install**

---

## 3.4 ドメインコントローラ昇格（既存ドメインに追加）

1. 通知フラグ → **Promote this server to a domain controller**
2. **Deployment Configuration**
   - ✅ **Add a domain controller to an existing domain**
   - Domain：`corp.local`
   - 認証：`CORP\Administrator`（またはドメイン管理者）で認証
3. **Domain Controller Options**
   - ✅ DNS
   - ✅ Global Catalog（両DCでGC有効化を想定）
   - DSRM パスワード設定 : addc01 と同一のパスワードを設定すること (→ 2.4節の管理方針参照)
4. **Additional Options / Paths**：既定のまま
5. **Prerequisites Check** → **Install**
6. 再起動

---

## 3.5 DNS設定（addc02：昇格後に最終化）

昇格後、設計どおり DC 自身参照にする：

- Preferred DNS：`172.16.1.49`（自身）
- Alternate DNS：`172.16.1.50`（addc01）

**確認**
```powershell
ipconfig /all
```

# 5. UPNサフィックス追加（corp.example.com）

`corp.local` は non-routable のため、ユーザーのサインインID整合性を考慮して `corp.example.com` を追加する。

---

## 5.1 追加手順（GUI）

1. addc01 で **Active Directory Domains and Trusts** を開く
2. 左ペイン最上位（Active Directory Domains and Trusts）を右クリック → **Properties**
3. **UPN Suffixes** に `corp.example.com` を入力 → **Add** → **OK**

---

## 5.2 動作確認（任意）

後続のユーザー作成時に UPN が選べること（例：`taro.yamada@corp.example.com`）
```powershell
Get-ADForest | Select-Object UPNSuffixes
```
期待値：`corp.example.com`が表示されること

---

## 7.1 テストユーザー作成（GUI）

1. `corp.local` → `SyncUsers` を右クリック → **New** → **User**
2. 例）
   - First name：`Taro`

   - User logon name：`taro.yamada`

3. パスワード設定 → **Finish**

## 7.2 テストグループ作成（GUI）

1. `SyncUsers` → 右クリック → **New** → **Group**

# 10. よくある失敗と対処

---

## 10.1 addc02 昇格時にドメインが見つからない

- **原因**：昇格前 DNS が addc01 を参照していない  
- **対処**：昇格前は addc02 の **Preferred DNS** を `172.16.1.50` に設定して再試行

---

## 10.2 レプリケーションエラーが出る

### 原因候補
- IP / GW / DNS 設定誤り（特に DNS）
- 名前解決ができていない（`nslookup addc01` などで確認）

### 確認
```powershell
nslookup addc01
nslookup addc02
```

## 10.3 DSRMパスワードを忘れた

- **原因**：DSRMパスワードは通常ログオンでは使用しないため失念しやすい
- **対処**：ドメイン管理者権限で以下を実行し、リセットする
```powershell
ntdsutil "set dsrm password" "reset password on server null" quit quit
```

> ⚠️ リセット後は必ずパスワード管理ツールへ即時保管すること。

---

## 10.4 addc02 昇格後にレプリケーションが完了しない

- **原因**：レプリケーションの初回同期が自動完了していない場合がある
- **確認**：
```powershell
repadmin /replsummary
```
- **対処**：手動で強制同期を実行する
```powershell
repadmin /syncall /AdeP
```
- **再確認**：
```powershell
repadmin /replsummary
```
期待値：失敗 0 件であること

---

## 10.5 UPNサフィックスが追加できない

- **原因候補**：
  - `Active Directory Domains and Trusts` ではなく `Active Directory Users and Computers` を開いている
  - 左ペイン最上位（ルートノード）ではなくドメイン名を右クリックしている
- **対処**：
  1. `Active Directory Domains and Trusts` を開く
  2. 左ペイン**最上位のルートノード**（`Active Directory Domains and Trusts` と表示されている行）を右クリック
  3. **Properties** → **UPN Suffixes** タブで追加

- **確認（PowerShell）**：
```powershell
Get-ADForest | Select-Object UPNSuffixes
```
期待値：`corp.example.com` が表示されること

---

## 10.6 NTP同期ずれによるKerberos認証失敗

- **原因**：Kerberos は時刻差 5分以上でエラーになる仕様
- **確認**：
```powershell
w32tm /query /status
```
- **対処**：PDC Emulator（addc01）で外部NTPと強制同期する
```powershell
w32tm /resync /force
```
- **addc02 側も同期確認**：
```powershell
w32tm /query /status
```
期待値：`Source` が `addc01` を参照していること
