## 1. 対象システム構成詳細

### 1.1 構成概要

- 単一フォレスト／単一ドメイン構成  
- サインイン方式：Password Hash Synchronization（PHS）  
- Seamless SSO：有効  
- OUフィルタ：SyncUsers OU のみ同期  
- Active / Staging 冗長構成（Activeは常に1台）  
- カスタム同期ルールなし（既定同期ルールを使用）  

### 1.2 サーバ構成

| サーバ名 | 役割 |
| -------- | ---- |
| aadc01 | Active Directory Domain Services / DNS |
| mec01 | Microsoft Entra Connect (Active) |
| mec02 | Microsoft Entra Connect (Staging) |
| app01 | 検証端末 |

詳細なOS・IP・役割設定は「3.サーバ詳細設計」を参照。

## 2. Active Directory 詳細設計

### 2.1 フォレスト／ドメイン

| 項目 | 規定値 |
| ---- | ------ |
| フォレスト数 | 1 |
| ドメイン数 | 1 |
| ドメインFQDN | corp.local |
| NetBIOS名 | CORP |
| フォレスト機能レベル | Windows Server 2016 以上 |
| ドメイン機能レベル | Windows Server 2016 以上 |
| FSMO配置 | addc01 |
| グローバルカタログ | 有効 |

### 2.2 UPN設計

| 項目 | 設定値 |
| ---- | ------ |
| 規定UPNサフィックス | corp.local (ADドメインFQDN) |
| 追加UPNサフィックス | corp.example.com (ルーティング可能) |
| 使用UPNサフィックス | corp.example.com (ユーザー割当用) |

> 注記：corp.localはnon-routable ドメインのため、Entra IDとの整合性を考慮し、ルーティング可能なcorp.example.comをユーザー割当用サフィックスとして使用する。

### 2.3 OU設計(同期対象)

| 項目 | 設定値 |
| ---- | ------ |
| 同期対象OU | OU=SyncUsers,DC=corp,DC=local |
| 同期対象オブジェクト | User / Group |
| 同期対象外 | 上記以外すべて |

## 3. サーバ詳細設計

### 3.1 共通設計 (addc01 / mec01 / mec02)

| 項目 | 設定値 |
| ---- | ------ |
| OS | Windows Server 2022 Standard |
| IPアドレス | 固定IP |
| IPセグメント | 固定IP |
| DNS | addc01 を参照 |
| 時刻同期 | ドメイン階層に従う |
| セキュリティ | TLS1.2 以上必須 |

> IPアドレスの具体値は別紙パラメータシートにて確定後、反映する。


### 3.2 addc01 (AD DS / DNS)

| 項目 | 設定値 |
| ---- | ------ |
| 役割 | AD DS / DNS |
| グローバルカタログ | 有効 |
| FSMO | 5ロール保持 |


### 3.3 mec01 (Entra Connect / Active )

| 項目 | 設定値 |
| ---- | ------ |
| ドメイン参加 | corp.local |
| Entra Connect モード | Active |
| Staging Mode | False |
| Schedular | 有効 |


### 3.4 mec02 (Entra Connect / Staging)

| 項目 | 設定値 |
| ----- | ------ |
| ドメイン参加 | corp.local |
| Entra Connect モード | Staging |
| Staging Mode | True |
| Scheduler | 有効(Exportは実施しない) |

### 3.5 app01 (検証端末)

| 項目 | 設定値 |
| ---- | ------ |
| OS | Windows 11 |
| 用途 | 同期・サインイン検証 |
